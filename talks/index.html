Ace an Education in TCP

Ace is a TCP server that was built with the design priority of having welcoming source code. I wanted to learn more about the network layer and how to work with erlang/OTP.

This talk shows the path from simple echo functionality to a robust general purpose server. The challenges and encountered along the way. And why Elxir was such a perfect fit for this project.

It may also be possible to present this or similar as an more technical introduction to why BEAM to other communities.

Application border control

Lets build programs that express the problem domain. This is one of the best ideas from Domain Driven Design. However how to we have a progam composed of rich objects that make use of libraries that don't have the same semantics. How do we prevent our application from getting bogged down in implementation details. This talk discusses the above and in particular tackles the most pernicious implementation detail of HTTP servers.

In many ways this is the extension of "building on domain concepts", although all necesary ground about value objects is covered.

Rails the library

Rails is part of your toolkit and yet it makes a lot of demands for your time. Not just building a CRUD application? This talk discusses putting Rails in the corner and letting your application be more than a Rails app.

The content of this talk is based on production applications that have followed this approach. The guidance for the development was from the ideas of bounded contexts and pluggable services from Domain Driven Design

Object Oriented programming, One more time.

Spoiler: Objects come in two flavours, Facts and Actors.

Functional Programming(FP) and Object Oriented Programming(OOP) are often presented as opposites. Alan Kay doesn't mention classes and yet these are the first part of any tutorial on OOP. Joe Armstrong the creator of erlang, a clearly functional language, describes it as perhaps the only object oriented language.

This talk tries to clear up some of the many issues around OOP. OOP makes everything an object. I will argue this is a valid and yet useless. By getting more specific about the types of objects that exist we reconcile our difference and move on to a glorious future, perhaps.
